# T1036 - Masquerading

Tactic: Defense Evasion

Adversaries name or place a malicious binary to look like a legitimate one -
copying `/bin/bash` to `/tmp/.X11-unix/.update` and running it, or renaming
a payload to match a real system process. The goal is to blend into normal
process listings and file paths so a human (or a naive detection rule)
skims past it.

Why it matters: masquerading is a force multiplier for every other
technique - a reverse shell or persistence entry that also masquerades is
much harder to spot in a quick triage pass.

How this EDR detects it: the `masquerading_binary_path` Sigma rule flags
processes whose executable path is in an unusual location (e.g. `/tmp`,
`/dev/shm`) for a name that normally lives in `/usr/bin` or `/bin`, or
whose hash doesn't match the expected system binary. Atomic Red Team
validation: T1036.005 (Match Legitimate Name or Location).
