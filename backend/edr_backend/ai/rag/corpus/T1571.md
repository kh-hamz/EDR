# T1571 - Non-Standard Port

Tactic: Command and Control

Adversaries communicate over a port that doesn't match the protocol
normally associated with it (e.g. a reverse shell over port 4444, or C2
traffic tunneled over a port normally used for something else). Combined
with a shell-spawning interpreter (T1059) making the outbound connection,
this is the classic signature of a netcat/bash reverse shell.

Why it matters: it's the point where an attacker gains interactive control
of the host. It's also one of the highest-confidence detections available,
since legitimate services rarely open reverse shells to arbitrary ports.

How this EDR detects it: the `reverse_shell` Sigma rule matches a process
whose command line indicates piping a shell to a network connection (e.g.
`nc -e /bin/sh <ip> <port>`, `bash -i >& /dev/tcp/...`). This is the one
rule wired to the auto-response playbook - a match auto-issues a
`kill_process` command for the offending pid, since the confidence is high
enough to act without a human in the loop.
