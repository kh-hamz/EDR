# T1505.003 - Server Software Component: Web Shell

Tactic: Persistence

Adversaries drop a script (PHP, JSP, etc.) on a web server that gives them
a remote command interface through normal HTTP requests. On Linux this
shows up in process lineage as the web server's worker process (e.g.
`www-data`) spawning a shell - something a web server has no legitimate
reason to do.

Why it matters: a web shell is both an initial-access artifact and a
persistence mechanism - it survives service restarts and gives the
attacker a foothold that looks like ordinary web traffic in most log
sources, which is why process-lineage detection (not just log parsing)
matters here.

How this EDR detects it: the `webshell_child_process` Sigma rule matches a
shell or interpreter process whose parent is a web server process
(nginx/apache/www-data), combined with correlation grouping it with
whatever the shell does next. Atomic Red Team validation: webshell atomics.
