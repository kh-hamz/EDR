# T1136 - Create Account / T1548 - Abuse Elevation Control Mechanism

Tactic: Persistence, Privilege Escalation

Adversaries create a new local user (often with uid 0, granting root
without needing `sudo`) or add themselves to a privileged group/sudoers
file, so they have a durable, "legitimate-looking" way back in that doesn't
depend on the original exploited process staying alive.

Why it matters: this is one of the most durable persistence mechanisms -
a backdoor account survives reboots, patching, and even killing every
malicious process, and it's easy for an analyst to overlook a plausible
username in a user list.

How this EDR detects it: the `uid_zero_backdoor_user` Sigma rule watches
osquery's users snapshot for any account other than root with uid 0, or
newly added accounts appearing between scheduled snapshots. Atomic Red Team
validation: T1136.001 (Local Account).
