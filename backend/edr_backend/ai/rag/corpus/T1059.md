# T1059 - Command and Scripting Interpreter

Tactic: Execution

Adversaries abuse command and script interpreters (bash, sh, python,
powershell, etc.) to execute commands, scripts, or binaries. On Linux this
most often shows up as a shell spawned by a process that should not be
spawning shells - a network-facing service (nginx, apache), a cron job, or
an application server - which is a strong signal of remote code execution
or a compromised service account.

Why it matters: this is usually the *first* observable step after initial
access or exploitation. Catching it early, before persistence or lateral
movement, gives the best chance of low-cost containment.

How this EDR detects it: the `shell_from_service` Sigma rule flags a shell
process (bash/sh) whose parent is a known service process. Atomic Red Team
validation: T1059.004 (Unix Shell) atomics.
