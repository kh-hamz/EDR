# T1543.002 - Create or Modify System Process: Systemd Service

Tactic: Persistence, Privilege Escalation

Adversaries create or modify a systemd unit file so their payload starts
automatically at boot, or restarts if killed. This is systemd's equivalent
of the cron persistence technique, and is often preferred on modern distros
because unit files can be made to look like legitimate system services.

Why it matters: like cron persistence, this survives process kills and
even reboots. It's frequently paired with a masqueraded binary path
(T1036) so the unit name and executable both look benign at a glance.

How this EDR detects it: the `systemd_persistence` Sigma rule watches
osquery's systemd_units snapshot for newly added or modified unit files,
especially ones pointing at binaries outside standard system directories.
